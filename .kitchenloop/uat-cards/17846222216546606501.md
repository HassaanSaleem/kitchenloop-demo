# User Test Card: 17846222216546606501

(Covers the iteration-1 notes/search/share hardening bundle: title validation,
search robustness, case-insensitive + tag search, and share existence checks.)

## Feature (one sentence)

Over the relay-notes HTTP API a user can create notes (with a required,
non-empty title), find them by any word in their title, body, or tags
regardless of letter case, and only ever mint a share link for a note that
actually exists.

## Prerequisites
- [ ] Node.js v20 or newer is available
- [ ] curl is available
- [ ] You are in the repository root (the folder that contains `package.json` and `src/`)

## Prerequisite Verification Commands
```bash
node --version          # should print v20.x or higher

```

## Steps

Each step below boots the real server on its own port using the documented
entry point (`node src/server.mjs`), makes one request, and shuts the server
down. Copy and run each block exactly as written.

### Step 1: Creating a note with no title is rejected
```bash
PORT=3541 node src/server.mjs >/dev/null 2>&1 &
SRV=$!; sleep 1
curl -s -o /dev/null -w "HTTP %{http_code}\n" -X POST http://127.0.0.1:3541/notes -d '{"body":"no title, should be rejected"}'
kill $SRV 2>/dev/null
```
**Expected exit code:** 0
**Expected output contains:** "HTTP 400"
**Expected output does NOT contain:** "HTTP 201"

### Step 2: Creating a note with an empty title is rejected

### Step 8: Sharing a real note succeeds
```bash
PORT=3548 node src/server.mjs >/dev/null 2>&1 &
SRV=$!; sleep 1
B=http://127.0.0.1:3548
ID=$(curl -s -X POST $B/notes -d '{"title":"Shareable note"}' | node -e 'let s="";process.stdin.on("data",d=>s+=d).on("end",()=>process.stdout.write(JSON.parse(s).id))')
curl -s -o /dev/null -w "HTTP %{http_code}\n" -X POST $B/notes/$ID/share
kill $SRV 2>/dev/null
```
**Expected exit code:** 0
**Expected output contains:** "HTTP 201"
